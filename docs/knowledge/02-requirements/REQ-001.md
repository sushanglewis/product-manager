# REQ-001：录音工具可用性改进

## 背景

Lincoln 工作流目前通过 `record-interview` CLI 录制访谈音频。用户使用多种 AI 编程/协作工具（Conductor、Claude、Codex、OpenCode 等），希望录音入口跨工具通用、终端原生、可全局安装。

## 问题

1. 触发方式不友好：必须记住复杂的 `record-interview` 命令和参数。
2. 录音状态不可见：录音过程中终端缺少明确的“正在录音”提示。
3. 集成位置不自然：希望在当前 AI 工具工作区的终端中直接触发录音。
4. 交互过于高调：现有体验对工作流造成干扰，需要更低调的终端内触发方式。

## 方案

开发名为 `lincoln` 的跨平台终端 TUI 工具，通过 npm 全局安装。用户在任意工作区终端输入 `lincoln` 即可启动录音 TUI，完成后产物与现有 `record-interview` / `process-interview` 工作流保持一致。

## 验收标准

- [x] 任意工作区终端输入 `lincoln` 启动录音，无需记忆复杂参数
- [x] TUI 显示动态“正在录音”状态（含时长）
- [x] TUI 提供清晰停止方式
- [x] 停止后自动生成标准 `recordings/` 与 `interviews/` 产物
- [x] 可选择是否立即触发 `process-interview`
- [x] 产物格式与 `record-interview` CLI 一致
- [x] 支持 macOS，优先支持 Conductor 工作区终端

## 非目标

- 不实现专业级多轨音频编辑
- 不实现实时字幕或实时转写
- 不替换或废弃 `record-interview` Python CLI
- 不构建原生 GUI 或浏览器插件
- 第一阶段不强制支持 Windows/Linux
