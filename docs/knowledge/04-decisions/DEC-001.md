# DEC-001：Lincoln TUI 技术方案决策

## 背景

需要为 Lincoln 工作流选择一个终端录音 TUI 的技术栈，并决定是否复用现有后端。

## 决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| TUI 框架 | `ink` | React for Terminal，生态成熟，Claude Code 等工具也在使用 |
| 录音后端 | 复用 `record-interview` Python CLI | 避免重复实现 ffmpeg 录音逻辑，降低风险 |
| 语言/运行时 | TypeScript / Node.js | 便于 npm 分发，与 ink 生态一致 |
| 配置格式 | YAML/JSON `~/.lincolnrc` | 人类可读，易于手动编辑 |
| 音频电平获取 | ffmpeg 实时分析 | 不引入额外 Node.js 音频依赖 |
| 停止后触发 | 打印命令，不嵌套 spawn `claude` | 避免嵌套 Claude Code 会话和权限问题 |

## 影响

- `tools/lincoln/` 采用 TypeScript + Ink + React 18
- `tools/record-interview/` 继续作为录音后端维护
- 停止录音后用户需手动在同一终端执行 `claude process-interview <sessionId>`
